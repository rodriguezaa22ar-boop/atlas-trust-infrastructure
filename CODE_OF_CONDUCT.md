# Code of Conduct

## Our Standard

Atlas is a public trust and reviewer surface for authorized security workflows.
Participation should protect the same values the project tries to prove:
clarity, scope, evidence, verification, retention, release trust, and operator
control.

Contributors are expected to:

- use respectful, professional language;
- keep discussions focused on evidence and reviewable claims;
- preserve the authorized-use boundary;
- use synthetic or metadata-only examples when public discussion is needed;
- keep known limitations visible when proposing stronger claims;
- report sensitive security issues through the security policy instead of
  public issues or pull requests.

Unacceptable behavior includes:

- harassment, threats, or personal attacks;
- publishing credentials, tokens, private keys, session cookies, customer data,
  raw target data, packet captures, request bodies, response bodies, raw
  prompts, raw model output, or private runtime evidence;
- encouraging unauthorized access, autonomous exploitation, persistence,
  destructive testing, credential spraying, denial-of-service workflows,
  stealth or evasion behavior, or out-of-scope target expansion;
- presenting Atlas as externally audited, certified, legally compliant,
  tamper-proof, safety-guaranteed, or model-correct without evidence from the
  appropriate external authority.

## Enforcement

Project maintainers may remove comments, issues, pull requests, or other
contributions that violate this code of conduct or the repository security
policy. Maintainers may also block participation when conduct risks the public
trust surface or exposes private evidence.

Security-sensitive reports should follow [SECURITY.md](SECURITY.md).
